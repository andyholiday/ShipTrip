Roadmap

3 Phasen — minimales Risiko

Jede Phase einzeln ship-bar · kein neuer Dependency · kein SwiftData-Schema-Break

Phase 1 — Token-Konsistenz (1–2 Tage)

StatsView StatCards auf oceanBlue/sunsetOrange/seaGreen/oceanLight umstellen · BarMark foregroundStyle auf .oceanBlue.gradient · Donut-Palette an expenseColor(for:) binden · DealRowView Discount-Badge von .red auf .sunsetOrange · Tausender-Format in .formatted(.number) wrappen.

Effort S · Risiko: minimal (nur Farb-Substitution)

Phase 2 — Strukturverbesserungen (3–4 Tage)

Statistik: Hero-Gradient-Strip (4 Kernwerte) + 2×2 Token-Grid darunter · Merkliste: besten Deal als Hero-Card (CruiseHeroCardView-Muster, adaptiert) + Datum-Chips · Karte: Rollen-Token statt routeColor, Legende native ScrollView, Pin-Rollen-Legende · Mehr: Profil-Header (navy→oceanBlue) + Icon-Kacheln.

Effort M · Risiko: mittel (neue Subviews, kein Schema-Break)

Phase 3 — Interaktions-Akzente (2–3 Tage)

Reisen: Port-Preview-Chips in Hero-Card (horizontal HStack aus route, max 4) · Karte: Chip-Switcher (Alle / Region-Filter per selectedCruise) · Merkliste: Sort-Header mit Menu-Picker (Preis/Datum) · Stats: horizontale Bar-Charts für Top-Länder als Swift Charts HorizontalBarMark.

Effort M · Risiko: gering (reine UI-Erweiterungen, bestehende Daten)
